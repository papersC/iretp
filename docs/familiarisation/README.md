# DLD Data Familiarisation — Output Pack

**RFP reference:** DLD-IRETP-2026-001 v1.3, §12.1
**Purpose:** Mandatory Phase-1 deliverable. Five output documents must be completed and signed off before any data pipeline development begins.

## Required outputs

| # | Document | Status | Owner | Approver |
|---|---|---|---|---|
| 1 | [Source System Inventory](01_source_inventory.md) | Template | Vendor data lead | DLD Data Liaison Officer |
| 2 | [Field Mapping Document](02_field_mapping.md) | Template | Vendor data engineer | DLD IT |
| 3 | [Data Quality Baseline Report](03_data_quality_baseline.md) | Template | Vendor data analyst | DLD Data team |
| 4 | [Calculation Rules Validation](04_calculation_rules.md) | Template | Vendor analytics lead | DLD Analytics team |
| 5 | [Historical Data Assessment](05_historical_data.md) | Template | Vendor data engineer | DLD Data team |

## Microsoft Fabric environment assessment (§12.1 v1.3 addition)

In addition to the source-system catalogue, the v1.3 RFP requires a formal assessment of DLD's existing Microsoft Fabric environment. This is captured in:

- [Fabric Environment Assessment](06_fabric_environment.md)

This output is consumed by the [Architecture Integration Map](../ARCHITECTURE_INTEGRATION_MAP.md).

## Sign-off gate

Per RFP §12.2, no public data publication is permitted until the **Pre-Publication Analytics Assessment** is signed off by the DLD Data Liaison Officer. This pack feeds that gate.
